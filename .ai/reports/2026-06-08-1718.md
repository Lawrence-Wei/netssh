# Checkpoint: Blocking TOFU Host Key Verification

## Summary

- Replaced the old unknown-host-key auto-accept flow with a blocking TOFU challenge.
- Added `ssh_host_key_decide` so the frontend can accept once, trust and remember, or reject a host key.
- Added a Netssh-local `trusted_host_keys` SQLite table instead of writing to the user's OpenSSH `known_hosts`.
- Merged Netssh-trusted host keys with OpenSSH known hosts during SSH connection setup.
- Fixed known_hosts host matching for comma-separated host entries and `[host]:port` entries.
- Added TerminalPane host-key challenge UI before `sshOpen` can miss the event.
- Added focused Vitest coverage for unknown and mismatch host-key challenges.
